# Mémoire durable

Garder seulement les informations stables qui rendent l'Allié meilleur dans le temps.

## Garder

- préférence de voix ou de style ;
- règle de décision ;
- offre, promesse ou positionnement ;
- information client importante ;
- méthode répétée ;
- leçon observée plusieurs fois.

## Ne pas garder

- détail ponctuel déjà expiré ;
- émotion du moment présentée comme vérité stable ;
- secret, mot de passe, clé API ou donnée bancaire ;
- information client sensible inutile.

## Workflow

1. Reformuler en une phrase claire.
2. Demander validation si cela change durablement le comportement de l'Allié.
3. Écrire dans `.ally-memory/long-term/`.
4. Utiliser un fichier simple : `preferences.md`, `offres.md`, `clients.md`, `regles.md` ou `methodes.md`.
